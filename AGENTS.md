# Design repository instructions

Before performing visual, interaction, frontend, or product-design work:

1. Read `DESIGN.md`.
2. Read `docs/DESIGN_SYSTEM.md`.
3. Read the relevant layout or motion reference for the task.
4. Use `.agents/skills/warm-paper-workbench/SKILL.md` for interface design or implementation.
5. Use `.agents/skills/improve-animations/SKILL.md` for motion audits and planning.

Treat accepted entries in `docs/DECISIONS.md` as normative. Preserve the system's warm paper palette, compact controls, integrated container chrome, scope-based control placement, task-specific density, progressive disclosure, accessibility, and fluid interaction rules unless the current brief explicitly overrides them.

When introducing a durable design rule, update the decision log, canonical documentation, executable tokens, affected skills, and changelog in the same change.
