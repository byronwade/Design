# Apple mobile and tablet base overlay

Use Apple familiarity, agency, direct manipulation, semantic system controls, and accessibility while preserving the product’s Warm Paper identity.

## Shared behavior

Use standard back behavior, safe areas, system text and editing, context menus, drag and drop, share and document pickers, permissions, sheets, popovers, keyboard avoidance, and system appearance. User-driven motion tracks input, remains interruptible, and carries velocity when physical continuity matters. Haptics reinforce meaningful state changes rather than every tap.

## Navigation and commands

Navigation bars establish hierarchy, title, back, and concise contextual actions. Toolbars hold current-view commands. Tab bars represent a small stable set of top-level destinations and never become a row of generic actions. Commands remain available through platform-appropriate menus and keyboard shortcuts where hardware supports them.

## Type, materials, and accessibility

Use Dynamic Type roles, optical sizing, semantic colors, and system materials where they improve hierarchy and adapt to appearance, contrast, transparency, and wallpaper. Support VoiceOver, Switch Control, Full Keyboard Access where available, Increase Contrast, Reduce Motion, Reduce Transparency, and sufficient target size. Custom controls expose equivalent traits and actions.
