# Layout archetypes

A shell defines persistent application structure. A layout archetype defines how the current task body allocates attention and space. Select one dominant archetype and combine only when responsibilities remain explicit.

## Centered action

Use for authentication, onboarding, agent welcome, focused setup, or one primary command. Target approximately 620–720px for the action stack; do not stretch a singular task across the work surface. Parallel suggestions may use a small compact group only when they are genuinely equal next steps.

## Document plus supporting pane

Use a 720–800px primary reading or configuration column, targeting approximately 760px, with an optional 320px compact inspector or 384px rich rail. Titles, summaries, descriptions, and narrative content remain mostly unboxed. Independent stateful modules may use contained surfaces. Convert the pane to a route, sheet, or overlay before the primary column falls below approximately 640px.

## Full operational canvas

Use available body width for lists, tables, queues, boards, timelines, reporting, editors, canvases, and monitoring. Preserve semantic column widths and deliberate local scrolling. Keep identity and controls stable while dense work owns the necessary scroll region.

## Feed or conversation

Use when chronological activity, discussion, or generated events are primary. The composer lives where new content enters the sequence. Preserve unread position, drafts, thread context, loading direction, and durable links. Conversation does not become the only representation of structured work or evidence.

## Settings and properties

Use persistent category navigation and a centered 620–680px form column on desktop, or a predictable property grid in an inspector. Constrained layouts turn category selection into navigation rather than compressing the form. Long forms keep a stable commitment region and unsaved-state communication.

## Split comparison

Use when comparing two artifacts or representations is itself the task. Make ownership, synchronization, scrolling relationship, active side, and differences explicit. Do not use equal splits merely to fill space; each side needs a distinct responsibility and useful minimum.

## Selection evidence

State the archetype, primary content measure, supporting regions, density zone, local scrolling, minimum useful dimensions, constrained transformation, and why a simpler archetype is insufficient.
