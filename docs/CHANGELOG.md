# Changelog

All notable design-system changes are recorded here.

## 0.2.0 — 2026-07-21

- Added **Calm Capability** as the product-design judgment layer above Warm Paper Workbench.
- Added eight experience values: consider the whole situation, make capability feel attainable, compose complexity into calm, craft the whole path, protect momentum, make trust tangible, start familiar and become unmistakable, and grow without drift.
- Added explicit tension resolutions for calm versus capability, compactness versus access, efficiency versus safety, automation versus authority, dimensionality, density, cards, and structured work.
- Established Shopify’s experience-value structure and Vercel’s executable interface-quality posture as the two core external models.
- Added a source map covering GitLab, Ant Design, Rauno Freiberg, VS Code, GitHub content design, ONS, Joshua David Thomas, Apple, Emil Kowalski, and the internal Warm Paper, Calm Workbench, and Velocity lineage.
- Added a normative motion contract covering purpose and frequency, immediate response, spatial causality, interruptibility, velocity and momentum, timing, performance, reduced motion, and agentic status integrity.
- Added executable Calm Capability UI review guidance with file-and-line severity output.
- Completed synchronized Codex and Claude `improve-animations` skills with an eight-category audit catalog and executor-ready plan template.
- Updated repository authority, agent entry points, accepted decisions, governance, and source navigation.

## 0.1.1 — 2026-07-21

- Added the comprehensive Calm Paper Workbench preference reference without changing the accepted Warm Paper Workbench canonical name.
- Preserved four shell profiles, focus-mode behavior, product and marketing patterns, the explicit do/don’t catalog, and the design-review scorecard.
- Added a machine-readable JSON token bundle and a starter CSS shell/component layer alongside the canonical CSS tokens.
- Added a detailed design and release checklist, distinct v1 source concepts, and a source-artifact coverage manifest.
- Recorded the canonical-versus-supporting-reference relationship in the design decisions.

## 0.1.0 — 2026-07-21

- Established Warm Paper Workbench as the living design system.
- Added the warm cream and paper color foundation.
- Added compact control sizing and warm tactile shadow recipes.
- Added integrated toolbar/body/footer container rules.
- Added stable-shell and task-specific layout archetypes based on Linear interface research.
- Added component, responsive, content, accessibility, and governance guidance.
- Added fluid-interaction principles covering immediate response, direct manipulation, interruptibility, spring motion, velocity handoff, momentum, spatial consistency, rubber-banding, and reduced-motion equivalents.
- Added Codex and Claude project skills for interface work and animation auditing.
