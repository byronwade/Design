# Calm Paper Workbench preference reference

Version 1.0.0 · Supporting reference · 21 July 2026

> **Core rule:** Keep location stable. Let the work change shape. Reveal depth beside the work without allowing it to dominate.

This reference preserves the comprehensive synthesis of recurring personal UI/UX preferences, including shell variants, focus mode, product and marketing patterns, explicit do/don’t rules, implementation quality gates, and the review scorecard.

## Repository status

The accepted repository system name remains **Warm Paper Workbench**. `DESIGN.md`, accepted entries in `docs/DECISIONS.md`, `docs/DESIGN_SYSTEM.md`, and `tokens/tokens.css` remain normative. This reference is additive: use it for more detailed preference resolution when the normative files are silent, particularly for deep workbenches, header/sidebar exceptions, marketing and dashboard patterns, agent experiences, and the exhaustive do/don’t catalog.

Where this reference conflicts with an accepted decision or canonical token, follow the accepted repository source and record any proposed change in `docs/DECISIONS.md`.
