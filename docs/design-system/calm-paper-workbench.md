# Calm Paper Workbench

Version 1.0.0 · Living specification · 21 July 2026

> **Core rule:** Keep location stable. Let the work change shape. Reveal depth beside the work without allowing it to dominate.

This is the canonical personal UI/UX design system. The full specification is divided into stable, reviewable sections so that layout, component, and policy changes can be updated independently without turning the repository into a single unmanageable document.

## Specification

1. [`calm-paper-workbench/01-foundations-and-layout.md`](calm-paper-workbench/01-foundations-and-layout.md) — purpose, design DNA, shell profiles, layout archetypes, tokens, typography, spacing, radius, and elevation.
2. [`calm-paper-workbench/02-components-patterns-and-accessibility.md`](calm-paper-workbench/02-components-patterns-and-accessibility.md) — components, product patterns, responsive behavior, and accessibility gates.
3. [`calm-paper-workbench/03-do-dont-shell-layout-and-visuals.md`](calm-paper-workbench/03-do-dont-shell-layout-and-visuals.md) — explicit shell, navigation, density, layout, and visual-foundation rules.
4. [`calm-paper-workbench/04-do-dont-content-components-and-interaction.md`](calm-paper-workbench/04-do-dont-content-components-and-interaction.md) — content, controls, components, interaction, and state rules.
5. [`calm-paper-workbench/05-do-dont-responsive-marketing-and-quality.md`](calm-paper-workbench/05-do-dont-responsive-marketing-and-quality.md) — responsive, accessibility, marketing, dashboard, implementation, and quality rules.
